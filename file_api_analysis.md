# Анализ обработки файлов в GreenTalk

## Общая архитектура

Процесс обработки файлов в GreenTalk имеет несколько компонентов:

1. **Загрузка файлов** - HTTP маршрут `/upload` для загрузки файлов на сервер
2. **Хранение файлов** - Файлы сохраняются в директориях `/static/uploads/` (оригиналы) и `/static/compressed_uploads/` (сжатые версии)
3. **Передача файлов** - Файловые сообщения отправляются через Socket.IO с использованием специального формата
4. **Отображение файлов** - Клиент отображает файлы разных типов по-разному (изображения, видео, аудио, документы)

## Формат файловых сообщений

Файловые сообщения имеют специальный формат: `FILE:тип:url`, где:
- `FILE:` - префикс, указывающий, что это файловое сообщение
- `тип` - тип файла (`image`, `video`, `audio`, `document`)
- `url` - путь к файлу на сервере (обычно `/static/uploads/...`)

Пример: `FILE:image:/static/uploads/1234abcd-5678-efgh.jpg`

## Поток данных при отправке файла

1. Клиент загружает файл через HTTP POST запрос на `/upload`
2. Сервер сохраняет файл, создает миниатюру (для изображений) и возвращает URL
3. Клиент формирует сообщение в формате `FILE:тип:url`
4. Клиент отправляет сообщение через Socket.IO с использованием события `send_private_message`
5. Сервер сохраняет сообщение в базе данных и пересылает его получателю
6. Получатель обрабатывает сообщение и отображает файл в зависимости от его типа

## Обработка разных типов файлов

### Изображения
- Поддерживаемые форматы: jpg, jpeg, png, gif, webp
- Для изображений создаются сжатые версии (миниатюры)
- Отображаются непосредственно в интерфейсе чата
- Возможно увеличение при клике

### Видео
- Поддерживаемые форматы: mp4, webm, mkv, mov
- Отображаются с помощью элемента `<video>` с элементами управления
- Автоматически устанавливаются атрибуты для управления воспроизведением

### Аудио
- Поддерживаемые форматы: mp3, wav, ogg, opus
- Отображаются с помощью элемента `<audio>` с элементами управления
- Отображается визуализация для треков

### Документы
- Поддерживаемые форматы: pdf, doc, docx, txt и др.
- Отображаются как ссылки для скачивания
- Показывается иконка типа документа

## Серверные маршруты для файлов

### `/upload` (POST)
- Принимает файл через форму
- Проверяет тип файла на допустимость
- Генерирует уникальное имя с использованием UUID
- Сохраняет оригинал и создает сжатую версию (для изображений)
- Возвращает JSON с информацией о файле и URL

### `/static/uploads/<filename>` (GET)
- Обслуживает оригинальные файлы

### `/static/compressed_uploads/<filename>` (GET)
- Обслуживает сжатые версии файлов
- Выполняет декомпрессию, если файл сжат одним из алгоритмов
- Устанавливает правильный MIME-тип

## Сжатие файлов

GreenTalk использует несколько методов сжатия:
- Для изображений: PIL компрессия, отмечается заголовком 'P'
- Нейросетевое сжатие для изображений, отмечается заголовком 'N'
- BZ2 сжатие для общих типов файлов, отмечается заголовком 'B'
- LZMA сжатие для общих типов файлов, отмечается заголовком 'L'

## Socket.IO события для работы с файлами

### `send_private_message`
- Используется для отправки как текстовых, так и файловых сообщений
- Для файловых сообщений используется формат `FILE:тип:url`

### `receive_private_message`
- Получает сообщения, в том числе файловые
- Клиент определяет тип сообщения и отображает соответствующим образом

## Клиентская обработка

Клиент обнаруживает файловые сообщения по префиксу `FILE:` и обрабатывает их особым образом:
- Разбивает сообщение на части по `:` для получения типа и URL
- Отображает файл в зависимости от его типа, используя `renderFileMessage`
- Создает уникальные ID для медиа-элементов
- Обеспечивает корректный MIME-тип на основе расширения файла

## Инструменты для тестирования

Мы создали два инструмента для тестирования API файлов:

### `file_upload_test.py`
Утилита для тестирования загрузки и отправки файлов:
```
python file_upload_test.py <имя_пользователя> <пароль> <получатель> [путь_к_файлу]
```

### `socketio_monitor.py`
Утилита для мониторинга всех событий Socket.IO:
```
python socketio_monitor.py <имя_пользователя> <пароль> [продолжительность_в_секундах]
```

## Безопасность

- Проверка типов файлов через расширение
- Генерация случайных имен файлов с помощью UUID для предотвращения конфликтов
- Использование `secure_filename` для предотвращения атак на файловую систему
- Хранение метаданных файлов в базе данных
